# 用語集

JSTQB Foundation Level で押さえておきたい重要用語をまとめます。用語の表記・定義はシラバス（V4.0）に準拠しています。レッスンの公開に合わせて随時追加していきます。

::: tip 使い方
各レッスンの「キーワード」表と合わせて確認すると、用語の関係を整理しやすくなります。より正確な定義は公式シラバスおよび ISTQB 用語集を参照してください。
:::

## テストの基礎

| 用語 | 説明 |
|------|------|
| テスト（testing） | ソフトウェアの品質を評価し、運用時の故障のリスクを下げるための活動全体。欠陥の検出だけでなく、要件を満たすことの確認や品質に関する情報提供も含む。 |
| デバッグ（debugging） | 故障の原因である欠陥を特定し、分析し、除去する開発活動。テスト（故障を見つける）とは別の活動。 |
| エラー（誤り、error） | 人が誤った結果を生み出す行為。欠陥の原因になる。 |
| 欠陥（defect、バグ） | 作業成果物に含まれる不備。実行すると故障を引き起こすことがある。 |
| 故障（failure） | コンポーネントやシステムが期待する結果を出さない事象。欠陥が実行されたときに現れる。 |
| 根本原因（root cause） | 問題を引き起こした最も根本的な要因。取り除けば同種の問題の再発を防げる。 |

## テストプロセスと活動

| 用語 | 説明 |
|------|------|
| テストベース（test basis） | テストケースを導き出すもとになる情報（要件・仕様・設計・ユーザーストーリーなど）。 |
| テストウェア（testware） | テスト活動で作成・使用する成果物（テスト計画書・テストケース・テストデータ・テスト結果など）。 |
| テストケース（test case） | 前提条件・入力・期待結果などで構成される、テストの実行単位。 |
| トレーサビリティ（traceability） | テストベースとテストウェアの対応関係を追跡できること。カバレッジの把握や影響分析に役立つ。 |

## テストレベルとテストタイプ

| 用語 | 説明 |
|------|------|
| テストレベル（test level） | テスト活動をまとめたグループ。コンポーネントテスト、コンポーネント統合テスト、システムテスト、システム統合テスト、受け入れテストがある。 |
| テストタイプ（test type） | 特定の品質特性に着目したテストのまとまり。機能テスト、非機能テスト、ブラックボックステスト、ホワイトボックステストなど。 |
| 確認テスト（confirmation testing） | 欠陥を修正した後、その欠陥が本当に除去されたかを確かめるテスト。 |
| リグレッションテスト（regression testing） | 変更によって、変更していない箇所に悪影響（デグレード）が出ていないかを確かめるテスト。 |
