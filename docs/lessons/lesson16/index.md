# lesson16: 境界値分析 — パーティションの端に潜む欠陥を狙う

## このレッスンで学ぶこと

- 境界値分析の考え方と、同値分割法との関係を理解する
- 境界値分析を適用できるパーティションの条件を説明できるようになる
- 2値境界値分析と3値境界値分析の違いを説明できるようになる
- 具体的な仕様から境界値を識別し、テストケースを導出できるようになる
- 境界値カバレッジの測定方法を理解する

## 境界値分析の考え方

境界値分析（boundary value analysis、BVA）は、同値パーティションの境界を確認することに基づいたブラックボックステスト技法です（テスト技法の分類は [lesson14](/lessons/lesson14/)）。

同値分割法（[lesson15](/lessons/lesson15/)）では、同じパーティションの中ならどの値を選んでも結果は同じとみなします。境界値分析はその考え方を拡張し、パーティションの端にある値を意図的に選びます。

::: info 境界値とは
順序づけできるパーティションの**最小値と最大値**が、そのパーティションの境界値です。
:::

境界に着目する理由は、開発担当者が境界値の扱いでエラーを起こしやすいためです。境界値分析で見つかる代表的な欠陥には、次のようなものがあります。

- 実装された境界が、意図した位置より上または下にずれている（例: 「100以下」とすべき判定が「100未満」になっている）
- 境界の処理そのものが実装から抜け落ちている

欠陥がこのように特定の場所へ集中しやすいことは、テストの7原則の「欠陥は偏在する」（[lesson03](/lessons/lesson03/)）とも符合します。

### 適用できるパーティションの条件

境界値分析は、順序性のあるパーティションにのみ使用できます。

- 数値や日付のように、値に大小関係があり順序づけできること
- 同じパーティションに属する2つの要素の間にある値は、すべて同じパーティションに属すること

たとえば「支払い方法の種類」のように順序のない分類には、境界値分析は適用できません。そうした対象には同値分割法（[lesson15](/lessons/lesson15/)）を使います。

## 2値境界値分析と3値境界値分析

シラバスは、境界値分析の2つのバージョンを扱っています。両者は、カバレッジ100%の達成に必要な、境界値ごとのカバレッジアイテムの数が異なります。

| 方式 | 境界値ごとのカバレッジアイテム | テストする値 |
|------|------|------|
| 2値境界値分析（2値BVA） | 2つ | 境界値と、隣接するパーティションに属する最も近い値（隣接値） |
| 3値境界値分析（3値BVA） | 3つ | 境界値と、その両隣の値 |

2値BVAでは、カバレッジアイテムはすべて境界値と隣接値です。隣り合うパーティションが連続している場合、ある境界値の隣接値は隣のパーティションの境界値と一致します。

3値BVAでは、境界値の両隣を必ずテストするため、境界値ではない値がカバレッジアイテムに含まれることがあります。

### 3値境界値分析の厳密さ

3値BVAは、2値BVAでは見落とす欠陥を検出できる可能性があるため、より厳密な方式です。

`x <= 10`（10以下）という判定を、誤って `x == 10`（10に等しい）と実装した場合を考えます。

- 2値BVAのテスト値は 10 と 11 です。どちらを入力しても期待通りの結果が返るため、この欠陥は検出できません
- 3値BVAでは 9 もテストします。9 は本来「10以下」として受け付けられるべき値ですが、誤った実装では条件を満たさないため、欠陥が故障として現れます

## ワーク例

境界値分析はK3（適用）レベルの技法です。具体的な仕様からテスト値を導出する手順を、例題で確認します。

::: info 例題の仕様
注文数の入力欄は、1以上100以下の整数を受け付ける。それ以外の整数にはエラーメッセージを表示する。
:::

### 手順1 パーティションへの分割

まず同値分割法（[lesson15](/lessons/lesson15/)）でパーティションを識別します。境界値分析は、この分割結果を出発点にします。

| パーティション | 範囲 | 種類 |
|------|------|------|
| P1 | 0以下の整数 | 無効パーティション |
| P2 | 1以上100以下の整数 | 有効パーティション |
| P3 | 101以上の整数 | 無効パーティション |

### 手順2 境界値の識別

各パーティションの最小値と最大値が境界値です。この例では次の4つを識別できます。

- P1 の最大値: 0
- P2 の最小値: 1
- P2 の最大値: 100
- P3 の最小値: 101

```text
   P1（無効）         P2（有効）           P3（無効）
... -2  -1  [0] | [1]  2 ... 99 [100] | [101] 102 ...

[ ] が境界値（0、1、100、101 の4つ）
```

P1 の下限と P3 の上限は仕様からは定まらないため、この例では扱いません。実務では、データ型の最小値・最大値などを境界として追加することもあります。

### 手順3 テスト値の選択

2値BVAでは、境界値ごとに「境界値そのもの」と「隣接パーティション側の最も近い値」を選びます。

| 境界値 | カバレッジアイテム |
|------|------|
| 0 | 0 と 1（P2側の隣接値） |
| 1 | 1 と 0（P1側の隣接値） |
| 100 | 100 と 101（P3側の隣接値） |
| 101 | 101 と 100（P2側の隣接値） |

重複を除くと、2値BVAのテスト値は 0、1、100、101 の4つです。

3値BVAでは、境界値ごとにその両隣の値も選びます。

| 境界値 | カバレッジアイテム |
|------|------|
| 0 | -1、0、1 |
| 1 | 0、1、2 |
| 100 | 99、100、101 |
| 101 | 100、101、102 |

重複を除くと、3値BVAのテスト値は -1、0、1、2、99、100、101、102 の8つです。2 や 99 のように境界値ではない値が含まれる点が、2値BVAとの違いです。

### 手順4 テストケースの導出

2値BVAのテスト値をテストケースに落とし込むと、次のようになります。

| テストケース | 入力値 | 期待結果 |
|------|------|------|
| TC1 | 0 | エラーメッセージを表示する |
| TC2 | 1 | 注文数として受け付ける |
| TC3 | 100 | 注文数として受け付ける |
| TC4 | 101 | エラーメッセージを表示する |

### 同値分割法との併用

境界値分析は単独で使うのではなく、同値分割法と組み合わせて使います。

- 同値分割法で、各パーティションの代表値（例: 50）が期待通りに扱われることを確認する
- 境界値分析で、パーティションの端（0、1、100、101）にずれや抜けがないことを確認する

パーティションの内部は同値分割法が、端は境界値分析がカバーするという役割分担です。

## 境界値カバレッジ

カバレッジの測定方法は、2値と3値で数え方が異なります。

- 2値BVA: テストで通した境界値の数を、識別した境界値の総数で割り、パーセンテージで表す
- 3値BVA: テストで通した境界値と隣接値の数を、識別した境界値と隣接値の合計数で割り、パーセンテージで表す

ワーク例で計算してみます。

| テストした値 | 2値BVAでのカバレッジ | 3値BVAでのカバレッジ |
|------|------|------|
| 0、1、100 | 4つ中3つで75% | 8つ中3つで37.5% |
| 0、1、100、101 | 4つ中4つで100% | 8つ中4つで50% |
| -1、0、1、2、99、100、101、102 | 100% | 8つ中8つで100% |

::: warning カバレッジの値は数え方に依存する
同じテストケースでも、2値で数えるか3値で数えるかでカバレッジの値は変わります。カバレッジを報告するときは、どちらの方式を採用したかを明確にします。
:::

## 試験のポイント

- 4.2.2はK3のため、与えられた仕様から「パーティション → 境界値 → テスト値」の順で自分で導出する適用問題に備える
- 境界値の数え方は「各パーティションの最小値と最大値」（例題では3つのパーティションから 0・1・100・101 の4つ）
- 2値境界値分析のカバレッジアイテムは境界値と隣接値で値の重複により総数が減り（例題では4値）、3値境界値分析は境界値の両隣も含むため境界値ではない値もカバレッジアイテムになる（例題では8値）
- カバレッジ計算の勘所は分母が方式で変わることで、同じテスト値でも2値と3値で結果が異なる（例題の4値は2値で100%、3値で50%）
