# lesson29: テストツールの種類と支援 — テスト活動を促進するツールの分類

## このレッスンで学ぶこと

- テストツールが支援する対象は、テスト実行の自動化だけではないことを理解する
- シラバスが挙げるテストツールの種類と、それぞれが支援するテスト活動を対応づけられるようになる
- ツールによる支援が特に効果を発揮する場面を理解する
- 表計算ソフトのような汎用ツールも、テストのコンテキストではテストツールになることを理解する

## テストツールによる支援の広がり

「テストツール」と聞いて思い浮かべやすいのは、テストを自動で実行するツールです。しかし、ツールが支援する対象はテスト実行に限りません。

テストツールによる支援は、多くのテスト活動を促進します。テスト計画からテスト完了までのテストプロセスの活動（[lesson04](/lessons/lesson04/)）に加えて、マネジメントやコミュニケーションもツールの支援対象です。

::: tip ツールが特に力を発揮する場面
- 反復的で単調な作業（リグレッションテストの繰り返し実行、同じテストデータの再入力など）
- 手動で行うことが困難、または不可能な作業（大量データの比較、性能テストでの負荷のシミュレーションなど）

人手ではミスが混入しやすい作業や、そもそも人手では実現できない作業ほど、ツールによる支援の価値が高くなります。
:::

## ツールの種類と支援する活動

シラバスは、テスト活動を促進するツールとして次の種類を挙げています。この分類に限定されるわけではありません。

| ツールの種類 | 支援する活動・もたらす効果 |
|------|------|
| マネジメントツール | SDLC、要件、テスト、欠陥、構成のマネジメントを容易にし、テストプロセスの効率性を向上させる |
| 静的テストツール | レビューや静的解析（[lesson11](/lessons/lesson11/)）を行うテスト担当者を支援する |
| テスト設計および実装ツール | テストケース、テストデータ、テストプロシジャーの作成を促進する |
| テスト実行とカバレッジツール | テスト実行の自動化とカバレッジの計測を容易にする |
| 非機能テストツール | テスト担当者が手動で行うことが困難、または不可能な非機能テストを実行できるようにする |
| DevOps ツール | DevOps デリバリーパイプライン、ワークフロー追跡、自動ビルドプロセス、CI/CD（[lesson07](/lessons/lesson07/)）を支援する |
| コラボレーションツール | コミュニケーションを円滑にする |
| 拡張性と展開の標準化をサポートするツール | 仮想マシンやコンテナ化ツールなど。環境の拡張や展開の標準化を支援する |
| その他のツール | 上記以外で、テストを補助するあらゆるツール |

### 種類と活動の対応づけ

学習目標（FL-6.1.1、K2）は「さまざまな種類のテストツールがどうテストを支援するかを説明する」ことです。ツールの種類と支援する活動の対応関係を押さえておきましょう。

- マネジメントツールが対象とする「構成のマネジメント」は、構成管理（[lesson27](/lessons/lesson27/)）を指します。要件・テスト・欠陥のマネジメントと合わせて、テストプロセスの効率性を高めます。
- 静的テストツールが支援するのは、レビューと静的解析という静的テストの活動です（[lesson11](/lessons/lesson11/)）。
- 非機能テストツールの代表例は性能テストツールです。多数の同時アクセスによる負荷は人手で再現できないため、ツールが不可欠です。
- DevOps ツールは、CI/CD や自動ビルドプロセスなど、DevOps（[lesson07](/lessons/lesson07/)）の実践を支えます。

## 汎用ツールもテストツールになる

テスト専用に作られたツールだけがテストツールではありません。表計算ソフトのような汎用ツールも、テストのコンテキストで使用すればテストツールになります。テストデータの整理やテスト結果の集計に表計算ソフトを使う場合、それはテストを補助するテストツールです。

::: info ツール導入は自動的に成功しない
ツールを導入しただけでは成功は保証されません。テスト自動化には利点だけでなく、分析と軽減が必要なリスクもあります。テスト自動化の利点とリスクは [lesson30](/lessons/lesson30/) で扱います。
:::

## キーワード

| 用語 | 説明 |
|------|------|
| テストツール（test tool） | テスト活動を支援、促進するツール。テスト専用ツールに限らず、テストのコンテキストで使う汎用ツールも含む |
| テスト自動化（test automation） | ツールを使用してテスト活動を実行または支援すること。利点とリスクは [lesson30](/lessons/lesson30/) で扱う |
| 静的解析（static analysis） | ツールを用いて、コードなどの作業成果物を実行せずに評価すること（詳細は [lesson11](/lessons/lesson11/)） |
| CI/CD | 継続的インテグレーションと継続的デリバリー。DevOps ツールが支援する（詳細は [lesson07](/lessons/lesson07/)） |
| コンテナ化ツール（containerization tool） | 拡張性と展開の標準化を支援するツールの例。仮想マシンと並んで挙げられる |

## 試験のポイント

- 学習目標は K2「さまざまな種類のテストツールがどうテストを支援するかを説明する」（FL-6.1.1）
- テストツールの支援対象はテスト実行の自動化に限らず、マネジメント・静的テスト・テスト設計と実装・非機能テスト・コラボレーションなど多くのテスト活動に及ぶ
- マネジメントツールは SDLC、要件、テスト、欠陥、構成のマネジメントを容易にし、テストプロセスの効率性を向上させる
- 静的テストツールはレビューと静的解析を、テスト設計および実装ツールはテストケース・テストデータ・テストプロシジャーの作成を支援する
- テスト実行とカバレッジツールは、テスト実行の自動化とカバレッジを容易にする
- 非機能テストツールは、手動で行うことが困難または不可能な非機能テストを可能にする
- DevOps ツールは、DevOps デリバリーパイプライン、ワークフロー追跡、自動ビルドプロセス、CI/CD を支援する
- 拡張性と展開の標準化をサポートするツールの例は、仮想マシンやコンテナ化ツール
- 表計算ソフトのような汎用ツールも、テストのコンテキストではテストツールになる
