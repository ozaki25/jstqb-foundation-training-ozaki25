# スプリントログ

アジャイルに進めるためのログ。1 スプリント = スコープ + 多視点レビュー + 修正 + 検証（lint / build / quiz:validate）+ コミット。

## レビュー観点（毎スプリントこの中から該当するものを適用）

1. **正確性**: シラバス V4.0 の用語・定義・K3 技法（境界値分析・デシジョンテーブル等）の計算例と矛盾しないか
2. **網羅性**: 担当範囲のシラバスの学習目標（LO）をすべてカバーしたか
3. **CLAUDE.md 準拠**: レッスン構成（学ぶこと→本文→キーワード→試験のポイント）・執筆スタイル
4. **用語統一**: シラバス表記の優先（「デシジョンテーブル」「欠陥/故障/エラー」「確認テスト/リグレッションテスト」等）
5. **読者目線**: テスト未経験の学習者が読めるか。専門用語の初出定義はあるか
6. **可読性**: 長い段落がないか。表・箇条書き・図への分解
7. **アクセシビリティ**: 図の alt、色だけに依存しない説明
8. **リンク整合**: 相互参照が `/lessons/lessonNN/` 形式か。リンク切れがないか
9. **ドリル品質**: 正解の正しさ・選択肢の質・解説とレッスンの一致・難易度/正解位置のバランス・K1〜K3 レベルとの対応
10. **ビルド健全性**: textlint / quiz:validate / docs:build がすべて通るか
