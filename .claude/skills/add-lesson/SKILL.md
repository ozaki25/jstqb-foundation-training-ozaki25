# レッスン追加スキル

JSTQB Foundation Level 学習コンテンツに新しいレッスンを追加します。
このプロジェクトはブランチ運用をせず、**main ブランチ上**で直接作業します。

## 引数

- `$ARGUMENTS` にレッスン番号・テーマ・シラバス該当箇所が渡されます
- 例: `/add-lesson lesson17 デシジョンテーブルテスト シラバス 4.3`

## 作業手順

以下のステップを**すべて**実行してください。

### 1. 現在のブランチ・最新化を確認

```bash
git status
git pull origin main
```

main ブランチ上で作業する。draft ブランチや `docs/drafts/` は使わない。

### 2. レッスン番号を確定

`docs/lessons/` 配下の既存レッスンと `docs/.vitepress/config.mts` のサイドバーを確認し、対象の `lessonNN`（NN はゼロ埋め2桁、lesson01〜lesson30）が未作成であることを確認する。

```bash
ls docs/lessons/
```

サイドバーには全30レッスンの章立て・仮タイトルが既に登録されている想定なので、対象の番号がどの章・どのテーマに割り当てられているかを `docs/.vitepress/config.mts` で確認する。

### 3. 必須インプット: シラバス該当箇所の確認

執筆前に**必ず**行うこと。省略しない。

- 対象レッスンが対応する **JSTQB/ISTQB シラバスの節番号・見出し**を特定し、該当箇所を通読する
- そのシラバス範囲に出てくる**重要ワード（用語）をすべて洗い出し、レッスン内で網羅**する。取りこぼした用語がないか、シラバスの目次・索引と突き合わせる
- 執筆中に**自信のない記述（シラバスの記述と微妙に異なる、複数解釈がありうる、最新版シラバスとの差分が不明な箇所など）があれば、書き上げる前に申告**する。曖昧なまま断定的に書かない
- 用語はシラバスの訳語をそのまま使う（独自の意訳をしない）。二次情報（解説記事・検索結果の要約）は参考にとどめ、表記の最終確認は必ずシラバス本文（一次情報）で行う

### 4. 既存レッスンの確認

同じ章の前後レッスンや、関連するテーマのレッスンを読んで、重複や整合性を確認する。
用語の説明が既存レッスンと食い違わないようにする。

### 5. レッスンファイルの作成

`docs/lessons/lessonNN/index.md` を作成します。

レッスンは以下の構成で、**15分程度で読み切れるボリューム**にしてください。

```markdown
# lessonNN: テーマ — サブタイトル

## このレッスンで学ぶこと

- 学ぶこと1（1行・1文。「〜を理解する」「〜を説明できる」など）
- 学ぶこと2
- 学ぶこと3
（3〜5項目）

## 本文（テーマに応じた見出しで複数セクション）

（シラバスの定義・仕組み・具体例を、仕組み・背景から説明する）
（読むだけで概念が伝わるように、文脈がわかる完結した例や表を載せる）
（デシジョンテーブル・状態遷移図などシラバスに図表の慣習がある場合はその形式に厳密に従う）

## キーワード

| 用語 | 説明 |
|------|------|
| 用語1 | シラバスに基づく簡潔な説明 |
| 用語2 | シラバスに基づく簡潔な説明 |

## 試験のポイント

- 出題されやすい観点1（定義の言い換え・混同しやすい用語の区別など）
- 出題されやすい観点2
（本文の繰り返しではなく、想起のトリガーになる短い箇条書き）
```

執筆方針:
- 見出しは名詞句で簡潔に。語りかけ調・詩的な語・「説明の足場」フレーズを避ける（詳細は review-lesson スキルのチェックリスト D・E を参照）
- 「このレッスンで学ぶこと」「試験のポイント」はそれぞれ **3〜5項目・各1行**に絞る。本文の説明を繰り返さない。項目が増えそうなときは、分割ではなく統合で数を減らす
- 用語・分類・図の構造はシラバスの表記に厳密準拠させる（デシジョンテーブルは条件/アクション×ルール列、状態遷移は状態・イベント・遷移・アクション、リスクは「リスクの可能性」×「リスクの影響」など）
- レッスン単体で完結させる。「前回」「lessonXX で」など他レッスンへの参照をしない

### 6. サイドバーの確認・更新

`docs/.vitepress/config.mts` の `themeConfig.sidebar["/lessons/"]` を確認する。

- 対象 `lessonNN` の項目が既に該当する章にプレースホルダーとして存在する場合は、テーマ名が実際に書いた内容と一致しているか確認し、必要なら文言を修正する
- 存在しない場合（章立てを変更した、または計画外のレッスンを追加した場合）は、適切な章の `items` に追加する

```typescript
{ text: "lessonNN: テーマ名", link: "/lessons/lessonNN/" },
```

### 7. 機械チェック・ビルド確認

以下をすべて実行し、通ることを確認する。

```bash
node .claude/hooks/check-lesson-title.js docs/lessons/lessonNN/index.md
node .claude/hooks/check-style.js docs/lessons/lessonNN/index.md
node .claude/hooks/check-bold.js docs/lessons/lessonNN/index.md
npm run docs:build
```

エラーや指摘があれば修正し、再度実行して確認する。可能であれば `review-lesson` スキルでレッスン単位のレビューも行う。

### 8. 完了報告

作成したファイル・更新したファイルの一覧と、実施したチェックの結果を報告する。
コミットするかどうかは、明示的な指示がない限りユーザーに確認する。
